# 制造业销售数据分析项目

## 项目背景
为中小型冲压件制造企业搭建销售数据分析体系，识别产品趋势，辅助业务决策。

## 核心成果

### 1. SQL分析体系
| 查询 | 用途 | 技术点 |
|:---|:---|:---|
| 年度/月度销售额统计 | 基础业绩追踪 | GROUP BY聚合 |
| Top10产品分析 | 识别核心贡献产品 | ORDER BY+LIMIT |
| 不同产品各月销售额汇总 | 产品维度透视 | 多字段分组 |
| 月度销售额环比变化 | 趋势分析 | 窗口函数LAG |
| 连续2个月销售额下滑预警 | 风险识别 | 窗口函数嵌套 |

### 2. PowerBI看板
- **基础版**：年度趋势、产品排名
- **预警版**：动态筛选、条件格式标红、单产品趋势追踪

## 技术栈
- MySQL 8.0（窗口函数、复杂子查询）
- Python pandas（数据清洗）
- PowerBI Desktop（可视化）

## 数据范围
- 时间跨度：2024年1月 - 2025年12月（24个月）
- 数据量级：500+条订单，100+SKU
- 核心指标：销售额、销售量、单价趋势

## 目录结构
├── data/                    # 原始数据及清洗结果
├── sql/                     # SQL查询文件
│   ├── 基础统计（年度/月度/Top10）
│   ├── 产品维度分析（各月汇总）
│   └── 趋势预警（环比/连续下滑）
├── power BI/                # PowerBI看板文件
└── reports/                 # 静态报表截图

## 关键发现
- 某主力产品2025年出现多次连续下滑，全年呈衰退趋势
- 部分产品存在明显季节性波动，Q4普遍承压
- 少数产品单价波动大，需关注定价策略

## 学习收获
- 掌握窗口函数实现时序分析（LAG/OVER）
- 理解B2B制造业数据特点：订单集中、波动大、客户依赖度高
- 建立"基础统计→趋势分析→风险预警"的完整分析链路
